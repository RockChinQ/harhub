# Release Notes

Use this skill when an agent needs to summarize completed changes into user-facing or internal release notes.

## Procedure

1. Group changes by feature, fix, operational change, and developer experience.
2. Prefer user impact over commit-by-commit narration.
3. Call out migrations, compatibility notes, and known risks.
4. Preserve exact version numbers, dates, and issue references when provided.

## Validation

- Notes are concise enough to scan.
- Risk and migration notes are not hidden inside generic bullets.
- Internal-only implementation details are excluded from user-facing sections.
